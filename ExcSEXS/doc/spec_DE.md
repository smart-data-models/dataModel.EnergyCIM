## Liste der Eigenschaften  

- `address`: Die Postanschrift  - `alternateName`: Ein alternativer Name für diesen Artikel  - `areaServed`: Das geografische Gebiet, in dem eine Dienstleistung oder ein angebotener Artikel erbracht wird  - `dataProvider`: Eine Folge von Zeichen, die den Anbieter der harmonisierten Dateneinheit identifiziert.  - `dateCreated`: Zeitstempel der Entitätserstellung. Dieser wird normalerweise von der Speicherplattform zugewiesen.  - `dateModified`: Zeitstempel der letzten Änderung der Entität. Dieser wird in der Regel von der Speicherplattform vergeben.  - `description`: Eine Beschreibung dieses Artikels  - `efdmax`: Maximaler Grenzwert der Feldspannungsbegrenzung (Efdmax).  Typischer Wert = 5. Voreinstellung: 0,0  - `efdmin`: Minimaler Grenzwert der Feldspannungsbegrenzung (Efdmin).  Typischer Wert = -5. Voreinstellung: 0,0  - `emax`: Maximaler Feldspannungsausgang (Emax).  Typischer Wert = 5. Voreinstellung: 0,0  - `emin`: Minimaler Feldspannungsausgang (Emin).  Typischer Wert = -5. Voreinstellung: 0.0  - `id`: Eindeutiger Bezeichner der Entität  - `k`: Verstärkung (K) (>0).  Typischer Wert = 100. Voreinstellung: 0.0  - `kc`: Verstärkung des PI-Reglers (Kc).  Typischer Wert = 0,08. Voreinstellung: 0,0  - `location`:   - `name`: Der Name dieses Elements.  - `owner`: Eine Liste mit einer JSON-kodierten Zeichenfolge, die auf die eindeutigen Ids der Eigentümer verweist  - `seeAlso`: Liste von uri, die auf zusätzliche Ressourcen über das Element verweist  - `source`: Eine Folge von Zeichen, die die ursprüngliche Quelle der Entitätsdaten als URL angibt. Empfohlen wird der voll qualifizierte Domänenname des Quellanbieters oder die URL zum Quellobjekt.  - `tatb`: Ta/Tb - Verhältnis der Verstärkungsreduzierung des Schleppabstandselements (TaTb).  Typischer Wert = 0,1. Voreinstellung: 0,0  - `tb`: Nenner Zeitkonstante des Schleppabstands-Blocks (Tb).  Typischer Wert = 10. Voreinstellung: 0  - `tc`: PI-Regler Phasenvorlaufzeitkonstante (Tc).  Typischer Wert = 0. Voreinstellung: 0  - `te`: Zeitkonstante des Verstärkungsblocks (Te).  Typischer Wert = 0,05. Voreinstellung: 0  - `type`: NGSI-Typ. Es muss ExcSEXS sein    
Erforderliche Eigenschaften  
Dieses Datenmodell ist eine direkte Umsetzung des Common Information Model (CIM), das durch die Norm IEC61970 spezifiziert ist, in Smart Data Models. Die Python-Klassen, auf denen dieses Modell basiert, wurden von diesen Einrichtungen entwickelt Institut für Automatisierung komplexer Stromversorgungssysteme (ACS), EON Energy Research Center (EONERC) und RWTH Aachen, Deutschland. einige Eigenschaften können einen falschen Typ haben. Dies war der Fall, bitte melden Sie einen Fehler oder senden Sie eine E-Mail an alberto.abella@fiware.org
